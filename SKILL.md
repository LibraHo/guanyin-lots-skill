# 观音灵签

## 原则

- 将求签定位为民俗仪式与反思工具，不宣称能确定预测未来。
- 一次只处理一个清楚问题。先询问问题、时间范围和必要背景；用户不愿透露隐私时允许概括表达。
- 先选起签方法和确认规则，再开始；过程中不得因结果不喜欢而更换规则。
- 一百签必须等概率。优先使用 `scripts/draw.py draw`；使用实体替代法时严格按 `references/drawing-methods.md`。
- 筊杯只负责确认是否继续或是否认可候选签，不用来改变签号。默认采用“连续三次圣筊确认”；若用户遵循特定寺庙规矩，以该规矩为准。
- 不把吉签解成保证，不把凶签解成灾难预告。医疗、法律、财务、安全事项必须建议依据现实证据和专业意见决策。

## 起签流程

1. 请用户默念或简述一个问题，限定一个主题和时间范围。
2. 让用户选择：系统公平抽取、秒表、七次硬币、两个十面数、实体百签。方法细节见 `references/drawing-methods.md`。
3. 若用户选择系统抽取，运行：

   ```bash
   python3 scripts/draw.py draw
   ```

4. 生成候选签后，以实体筊杯、两枚硬币或下列命令逐次确认：

   ```bash
   python3 scripts/draw.py cup
   ```

5. 严格三圣筊规则：连续三次圣筊才确认；任何笑筊或阴筊都使当前候选签失效，重新等概率抽取。若在“是否允许求签”阶段出现笑筊，先澄清问题再掷；出现阴筊则停止本次起签。
6. 单圣筊规则：一次圣筊即确认；笑筊先澄清再确认，阴筊停止或重新起签。开始前必须说明使用此规则。
7. 最多尝试合理次数。连续多次未确认时，建议停止，不为了得到满意结果反复抽取。

候选签每次均为 1–100 的独立均匀抽取，且确认概率不依赖签号，因此最终确认的每一签仍等概率。不得用语言模型“随口选数”代替随机过程。

## 查签

完整百签资料位于 `references/lots.json`。按签号查询，不要一次加载整份文件：

```bash
python3 scripts/draw.py lookup 42
```

资料保留繁体底本文字。面向简体中文用户时转为简体输出，同时注明明显异文；不要擅自把其他寺庙版本混入当前签文。版本和来源见 `references/sources.md`。

## 解读结构

阅读 `references/interpretation-guide.md`，然后按以下顺序回答：

1. 签号、签题、原签诗和底本标注的等级。
2. 逐句白话：解释意象、转折和条件，不只给“吉/凶”。
3. 核心判断：当前处境、主要阻力、可利用条件、建议行动。
4. 结合用户所问领域解读；只展开相关领域。
5. 给出一项现实可执行建议，以及一项需要避免的行为。
6. 简短说明这是传统文化解释，重要决定仍需现实信息验证。

若用户直接报签号，跳过随机与筊杯流程，直接查签解读。若用户提供的是另一版本签诗，以用户提供的原文为准，并说明与本技能底本的差异。
